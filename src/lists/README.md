# Часть 2 — Алгоритмы обработки списков и хеширование

Задачи 2.1–2.5 из методички [docs/lists.pdf](../../docs/lists.pdf).
Списки используются как стеки и очереди, реализованные на односвязном списке
с указателями `head` и `tail`.

## Используемые модули

- [lib/list.h](../../lib/list.h) — `List`, `StrList`, `MergeSort`, `DigitalSort` (int/int16/int32), `StrListDigitalSort`.
- [lib/hash.h](../../lib/hash.h) — `HashChain` (chaining), `HashOpen` (linear/quadratic probing).
- [lib/sorting.h](../../lib/sorting.h) — используется в `digital` для сравнения с `HeapSort`/`QuickSort`.

## Особенности

- **DigitalSort, теория M.** Реализация в [lib/list.c](../../lib/list.c) считает 4 пересылки на элемент за байтовый проход
  (список → корзина и корзина → список, по 2 каждая) ⇒ `M = bytes·4·n`.
  Для `int` (4 байта): `M = 16n`. Совпадение с фактом точное.

- **Сортировка фамилий по русскому алфавиту.** `StrListDigitalSort` использует
  байтовый LSD-radix; для UTF-8 кириллицы байтовый порядок совпадает с алфавитным А–Я
  (буква «ё» попадает в конец — стандартная компьютерная коллация).

- **Хеширование, m < n.** В 2.5 размер таблицы (m=11) намеренно меньше количества
  ключей (n=12), чтобы наглядно показать переполнение и поведение поиска (`-1`).

- Для **2.2** и **2.3**: главная ТЗ-таблица + `=== Доп. ===` с разбором M, C

- Для **2.1, 2.4, 2.5**: реальные данные (содержимое стека/очереди/таблиц)
  плюс сводные таблицы по списку простых чисел / типам пробирования.
